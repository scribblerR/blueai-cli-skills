# 输入解析与转换规则

> 把用户的中文/自然语言输入转换成 CLI `--params` 里的 API 原始值。**Step 2 渲染参数确认表前必须读取本文件。**
>
> 转换原则：
> - 全部转换在展示参数确认表之前完成
> - 确认表默认只两列：`参数 | 值`，使用用户可读标签，不展示 API 字段名和枚举数字
> - 用户开启 debug 模式时再加一列 `API 值`
> - 不确定的字段在"值"列用 `⚠️ 请核对` 标注

## 1. 日期 → Unix 时间戳（秒）

时区固定为 **Asia/Shanghai（UTC+8）**。

| 字段 | 边界规则 | 示例 |
|---|---|---|
| `start_time` | 当天 00:00:00 CST | `2026-01-01` → `1767196800` |
| `end_time` | 当天 23:59:59 CST | `2026-03-31` → `1774972799` |

**自然语言识别：**

| 输入 | 解析 |
|---|---|
| 今年1月1日 / 2026年1月1日 | 2026-01-01 |
| 上个月1号 | 当前月-1 的 1 号 |
| 最近3个月 | start=今天-90天，end=今天 |
| 2025年Q4 | start=2025-10-01，end=2025-12-31 |
| 去年全年 | start=2025-01-01，end=2025-12-31 |

## 2. 平台名 → media_id（仅 stock 模式）

读取 `references/platforms.json` 的 `stock_platforms`，按 aliases 匹配（大小写不敏感）：

| 输入 | media_id |
|---|---|
| 小红书 / xhs / 红书 / 小红薯 | 4 |
| 抖音 / douyin / dy | 5 |
| B站 / b站 / 哔哩哔哩 / bilibili | 7 |
| 微博 / weibo | 1 |

## 3. stock 模式枚举字段

**data_type（要不要评论）**

| 输入 | API 值 |
|---|---|
| 仅帖子 / 不要评论 | 0 |
| 帖子和评论 / 含评论 | 1 |

**search_type（匹配方式）**

| 输入 | API 值 |
|---|---|
| 精准 / 精确 | 0 |
| 模糊 / 分词 | 1 |

**match_fields（匹配字段，多选）**

| 输入 | API 值 |
|---|---|
| 标题 | "title" |
| 正文 / 内容 | "content" |
| 话题 / 标签 | "topic" |

**post_types（帖子类型，多选）**

| 输入 | API 值 |
|---|---|
| 普通视频 / 视频 | "video" |
| 普通图文 / 图文 | "image_text" |
| 广告视频 | "ad_video" |
| 广告图文 | "ad_image_text" |

> 用户说"排除广告" → `["video", "image_text"]`。

**sort（排序对象）**

```json
{"field": "interaction_cnt", "order": "DESC"}
```

| 输入字段 | API 值 | 输入方向 | API 值 |
|---|---|---|---|
| 互动量 / 热度 / 热门 | "interaction_cnt" | 降序 / 从高到低 / 最多 | "DESC" |
| 发布时间 / 最新 / 最早 | "publish_time" | 升序 / 从旧到新 | "ASC" |
| 点赞 / 点赞数 | "like_cnt" | | |
| 评论 / 评论数 | "review_cnt" | | |
| 转发 / 分享 | "reposts_cnt" | | |
| 阅读 / 浏览 | "view_cnt" | | |
| 收藏 / 收藏数 | "fav_cnt" | | |

## 4. 数量/阈值识别

| 输入 | 解析 |
|---|---|
| 最多500条 / 取前500 | `max_data_cnt=500` |
| 互动量大于1000 / 1k互动以上 | `min_interact_cnt=1000` |
| 互动1万+ / 爆款 | `min_interact_cnt=10000` |
| 正文不少于100字 | `min_content_len=100` |

## 5. 关键词格式化（stock 模式）

`+` 表示且，`|` 表示或：

| 输入 | 格式化 |
|---|---|
| 华为或小米的手机 | `华为\|小米+手机` |
| 防晒霜、防晒乳、防晒喷雾 | `防晒霜\|防晒乳\|防晒喷雾` |
| 华为手机并且是折叠屏 | `华为+手机+折叠屏` |

> 格式化结果必须在确认表中展示给用户复核。

# 公众号（realtime_wp）专用

## 6. 公众号标识识别

| 输入 | 识别为 | 字段 |
|---|---|---|
| 人民日报、新华社 | 公众号名称 | `names: [...]` |
| MzEzMDI...（Base64 形态） | biz ID | `bizs: [...]` |
| `https://mp.weixin.qq.com/s/...` | 文章 URL | `urls: [...]` |

多个标识用中/英文逗号或换行分隔自动拆分。同时提供多种类型时，API 优先级 `bizs > urls > names`，确认表中标注实际使用类型。

## 7. data_range（公众号采集范围）

| 输入 | API 值 |
|---|---|
| 历史 / 全部 / 所有文章 / 近期 | 1 |
| 今天 / 今日 / 当天 | 2 |

## 8. data_type（公众号详细程度，注意与 stock 不同语义）

| 输入 | API 值 | 说明 |
|---|---|---|
| 列表 / 简要 / 不含正文 | 1 | 仅标题/摘要 |
| 详情 / 含正文 / 完整 | 2 | 包含 content 字段 |

## 9. start_time（公众号采集开始）

仅 `data_range=1` 有效。规则与 stock 的 start_time 相同（00:00:00 CST → 时间戳）。

| 输入 | 解析 |
|---|---|
| 不填 / 默认 | 不传，API 默认最近 30 天 |
| 2026-01-01 | 1767196800 |
| 最近3个月 | 今天-90天的 00:00:00 CST |

最多近一年。

## 10. max_data_cnt（公众号每号最大文章数）

仅 `data_range=1` 有效。

| 输入 | 解析 |
|---|---|
| 每个号最多100篇 | `max_data_cnt=100` |
| 不填 | 不传，全部文章 |

# 公众号文章详情（realtime_wp_detail）专用

## 11. URL 解析与 mode

```
urls = [自动从用户消息中提取的所有 mp.weixin.qq.com 链接]
```

不含 `mp.weixin.qq.com` 的 URL 在确认表用 `⚠️ 请核对` 标注。

| mode 输入 | API 值 | 说明 |
|---|---|---|
| 带图片 / 含图片标签 | 1 | 纯文本，保留图片位置标签 |
| 纯文字 / 富文本 / 默认 | 2 | 纯文字 + 富文本 (content_multi_text) |

# 视频号（realtime_wx_channels_*）专用

## 14. 视频号名称 / 视频 ID 解析

| 输入 | 字段 |
|---|---|
| 人民日报、央视新闻 | `names: ["人民日报", "央视新闻"]` |
| `vid_abc123` 或 `wxv_xxx` | `video_ids: ["vid_abc123"]` |

逗号 / 空格 / 换行分隔自动拆分。

## 15. 搜一搜（web-search-videos）专用枚举

**sort_type**

| 输入 | API 值 |
|---|---|
| 综合 / 默认 | 0 |
| 最新 | 1 |
| 最热 / 热门 | 2 |

**publish_time_type**

| 输入 | API 值 |
|---|---|
| 不限 / 全部 | 0 |
| 一天内 / 最近1天 | 1 |
| 一周内 / 7天内 | 2 |
| 半年内 / 近半年 | 3 |

**sub_search_type**

| 输入 | API 值 |
|---|---|
| 不限 / 默认 | 0 |
| 仅账号 | 1 |
| 最新发布 | 2 |
| 朋友赞过 | 3 |
| 最近看过 | 4 |

## 16. is_download / need_gs_url（视频号）

| 输入 | 值 |
|---|---|
| 是 / 要下载链接 / 含下载 | true |
| 否 / 不要 / 默认 | false |

`need_gs_url` 同样规则。

## 17. start_time（视频号视频列表）

与 stock 的 start_time 相同（00:00:00 CST → 时间戳）。不填则获取全部可用数据。
